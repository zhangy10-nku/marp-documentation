Good Software Design - Kevin's Personal Rules for High-Quality Softwware


1. If you can't automate the quality checking and testing of your project, it should not go beyond a development environment.

2. Merge often. Review often. Refactor often.

3. MVP ASAP.

4. Bounce ideas. Be social. Seek to understand different perspectives and what problems they solve.

5. Talk with users often. Get feedback as soon as possible. Be responsive. Understand needs.
